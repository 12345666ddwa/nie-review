## 🎯 功能特色

- **交互式自测**：每个知识模块配有折叠练习题，先做题再看答案
- **题型预测**：每个知识点标注可能出现的考试题型
- **易错陷阱**：12个高频判断题陷阱，红色警告框标注
- **关键字高亮**：核心术语蓝色、易错词红色，防止走神
- **论述题标答**：四步框架（定义→观点→理解→案例）
- **图形综合分析题**：SVG供求曲线图+10空填写
- **响应式设计**：手机电脑均可阅读

## 🔧 制作工作流

```
课程PPT ──→ python-pptx提取考点清单
                    ↓
课堂录音转写 ──→ 交叉对比(100%覆盖验证)
                    ↓
              HTML单文件构建
                    ↓
        自测系统 + 美工设计 + 部署
```

### 一、输入提取规则

- 用 `python-pptx` 遍历每页每个shape（文本框+表格），保留层级信息
- PPT列出的知识点就是考试范围，不能自行删减
- 课堂笔记是扩充不是替代，录音转写中的人名需交叉核实

### 二、交叉对比规则

- 用脚本逐条搜索关键字，输出覆盖率报告（目标100%）
- PPT有笔记没有→补充 / 笔记有PPT没有→扩充保留 / 术语冲突→以教材为准

### 三、内容组织规则

每个知识点必须包含四要素：
1. **定义/内容**（零基础起步）
2. **题型预测标签**（分类→多选、定义→名词解释、是非→判断）
3. **易错陷阱**（trap-box红色警告）
4. **自测练习**（折叠题目+折叠答案，分布均匀）

### 四、自测系统规则

- 两层折叠：展开看题→再展开看答案
- 考卷格式：标注题型、选项ABCD
- 答案含解析：说明为什么对/错
- 分布均匀：每个知识模块都有
- 论述题按四步框架提供标答范文
- 综合分析题含SVG图形

### 五、美工设计规则

- 内容为王，美工不抢戏
- 明亮配色（蓝/绿/红/橙/紫分角色）
- 导航放顶部（粘性磨砂条）
- 动效适度（滚动入场/悬停上浮/折叠动画）
- 关键字高亮（蓝=核心、红=易错、黄底=重点句）
- 响应式布局适配手机

### 六、署名与部署规则

- 标注产权署名 + 学校/课程/学期信息
- 单HTML文件，零外部依赖
- GitHub Pages部署，base64大文件用@file上传
- 更新文件需带当前SHA

## ⚠️ Pitfalls

1. PPT提取不能只看标题，知识点藏在正文和表格里
2. 课堂人名听音误，必须核实学术人名
3. 自测题分布要均匀，不能有的有有的没有
4. 美工方向先确认再动手，避免大改
5. base64>100KB不能放命令行，必须用文件传递
6. GitHub更新需要SHA，否则409冲突
7. 覆盖率必须脚本验证，人肉检查必遗漏
